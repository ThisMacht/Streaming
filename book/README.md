# Lab 04 Jupyter Book

Start with `intro.md`, then read `architecture.md` and Tasks 1–6 in order. The pipeline notebook
collects executable intermediate outputs, and `conclusion.md` summarizes results and limitations.

Evidence is kept close to the report:

- `logs/`: terminal, connector, database, replay, checkpoint, and test output;
- `kafka/`: captured Kafka message keys and values;
- `images/`: architecture, Neo4j, MongoDB, and Spark figures.

The table of contents in `_toc.yml` defines the GitHub Pages reading order. This landing note is an
evidence-directory guide and intentionally does not duplicate the full report.
